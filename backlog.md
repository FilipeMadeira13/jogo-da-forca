# Backlog de melhorias - Jogo da Forca

Este arquivo reúne ideias de melhorias para o projeto, organizadas por prioridade e foco.

## Prioridade alta

- [x] Adicionar um menu inicial com opções como: jogar, ver regras e sair
- [x] Evitar letras repetidas, mostrando uma mensagem clara quando o jogador tentar usar a mesma letra novamente
- [x] Melhorar a validação de entrada para aceitar apenas letras válidas
- [x] Permitir reiniciar uma nova partida sem fechar o programa
- [x] Mostrar a palavra secreta ao final da partida de forma mais visual e amigável

## Prioridade média

- [ ] Adicionar níveis de dificuldade: fácil, médio e difícil
- [ ] Criar categorias de palavras, como animais, objetos, programação, frutas e países
- [ ] Implementar um sistema de pontuação por rodada vencida
- [ ] Adicionar um placar com os melhores resultados
- [ ] Incluir uma opção para jogar novamente após o fim da partida
- [ ] Melhorar a interface do terminal com mensagens mais bonitas e organizadas

## Prioridade baixa

- [ ] Adicionar suporte para palavras com acento e caracteres especiais
- [ ] Criar um modo de jogo com tempo limite para aumentar o desafio
- [ ] Salvar o histórico de partidas em um arquivo
- [ ] Adicionar sons ou animações simples no terminal
- [ ] Criar uma versão com interface gráfica usando Tkinter

## Melhorias de organização e qualidade

- [ ] Refatorar o código para deixar a estrutura mais limpa e fácil de manter
- [ ] Separar a lógica do jogo em funções e classes mais bem organizadas
- [ ] Adicionar comentários no código para facilitar futuras alterações
- [ ] Criar testes automatizados para validar o funcionamento do jogo
- [ ] Adicionar um arquivo de licença e mais detalhes de documentação

## Ideias extras

- [ ] Permitir escolher a linguagem do jogo
- [ ] Adicionar um modo multiplayer local
- [ ] Incluir dicas para cada palavra secreta
- [ ] Criar um modo de dificuldade personalizada pelo usuário

## Sugestão de ordem para implementar

1. Menu inicial e reinício de partida
2. Validação de entrada e tratamento de letras repetidas
3. Níveis de dificuldade e categorias
4. Pontuação e placar
5. Melhorias visuais e estrutura do código
